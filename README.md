# Sky Rush v0.1

オンライン登山アクションゲームのv0.1プロトタイプです。

## 実装済み

- 固定パスワード `progress4649` によるログイン
- 部屋作成、部屋一覧、部屋参加
- 作成者のみ開始できる待機ルーム
- Socket.IOによるリアルタイム同期
- Phaser 3の縦スクロール登山ステージ
- 左右移動、長押しジャンプ、壁ジャンプ、踏み台ジャンプ
- 足場上面/下面の当たり判定
- ゲーム開始時に20人までCPUプレイヤーを自動補充
- プレイヤー同士の押し合い判定
- 下は広く、上に行くほど狭くなるコース
- ゴール、勝敗判定、リザルト表示
- スタート前カウントダウン
- ジャンプ台
- チーム登山用の味方踏み台強化、チーム専用ジャンプ台、高壁エリア
- 他プレイヤーの頭への着地と踏み台ジャンプ
- 伸縮する可変長バー
- ラストスパート演出
- 押し出し時の軽いエフェクト
- Docker向け単一コンテナ構成

## ローカル起動

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開きます。複数タブでログインすると複数人プレイを試せます。

## Docker起動

Dockerが使える環境では、以下で単一コンテナとして起動できます。

```bash
docker compose up --build
```

## デプロイについて

AWS CLIとDockerが使える環境では、PowerShellからECS/Fargateへデプロイできます。

```powershell
.\scripts\deploy-ecs.ps1
```

このスクリプトはECRリポジトリ作成、Dockerイメージのビルドとpush、ECS/Fargate、ALB、VPC、CloudWatch Logsの作成/更新を行います。デフォルトでは日本からの遅延を抑えるため `ap-northeast-1` に `sky-rush-ecs` スタックを作成します。

削除する場合:

```powershell
.\scripts\delete-ecs.ps1
```

ECRリポジトリも含めて削除する場合:

```powershell
.\scripts\delete-ecs.ps1 -DeleteEcr
```

v0.1はサーバメモリ管理です。複数台構成にする場合、同じ部屋の参加者が別サーバーへ分散しないようにスティッキーセッションを有効化するか、Redisなどの共有状態管理を追加してください。

## 操作

- `A` / `D` または `←` / `→`: 左右移動
- `Space` を離す: ジャンプ
- `Space` 長押し後に離す: 高めにジャンプ
